# Architecture

## Directories

- **cmd/feedium** — точка входа приложения. Содержит `main.go` с функцией `main()`, которая инициализирует зависимости и запускает сервис.

- **internal/app** — доменный слой: сущности, интерфейсы, use-case'ы (сервисы), валидация. Каждый модуль живёт в собственном подпакете (например, `internal/app/source/`). Не зависит от инфраструктуры (БД, транспорт).

- **internal/app/\<module\>/adapters/** — адаптеры модуля. Реализации инфраструктурных интерфейсов, определённых в доменном слое. Например:
  - `adapters/postgres/` — GORM-реализация репозитория
  - `adapters/connect/` — Connect-go хендлер (транспортный слой)

- **internal/platform/** — инфраструктурные утилиты без бизнес-логики:
  - `platform/logger/` — фабрика `Init()`, возвращающая `*slog.Logger`
  - `platform/postgres/` — фабрика `Open(dsn)`, возвращающая `*gorm.DB`

- **internal/bootstrap** — код инициализации приложения: сборка зависимостей, конфигурация, lifecycle-хуки. Вызывается из `cmd/feedium/main.go`.

- **api** — описание внешнего API: proto-файлы и сгенерированный код (Connect-go). Служит контрактом между клиентом и сервером. Сгенерированный код не редактируется вручную.

- **migrations** — SQL-миграции базы данных (goose format). Файлы применяются последовательно и не редактируются после применения в production.

## Package Interaction Rules

1. Зависимости направлены строго по слоям:
   - `cmd` импортирует `bootstrap`
   - `bootstrap` импортирует `platform`, `app`, адаптеры
   - Адаптеры (`adapters/*`) импортируют свой доменный модуль из `app` и `platform`
   - `app` (доменный слой) не импортирует адаптеры
2. Интерфейсы объявляются в пакете-потребителе (в `app`, не в адаптерах).
3. Запрещено создавать интерфейсы без использования.
4. Доменные модули внутри `internal/app` не импортируют друг друга напрямую. Взаимодействие между модулями происходит через `internal/bootstrap`.
5. Пакеты `internal/platform/*` предоставляют инфраструктурные утилиты и не содержат бизнес-логики. Любой пакет внутри `internal` может импортировать `platform`, но `platform` не импортирует другие `internal/*` пакеты.
6. Пакет `api` не импортирует `internal/*`. Сгенерированный код из `api` используется в `internal` как зависимость, но не наоборот.
7. `cmd/feedium` содержит только вызов bootstrap и минимальный glue-код. Бизнес-логика, конфигурация и инициализация зависимостей выносятся в `internal/bootstrap`.

## How to Add a New Module

1. Создайте подпакет `internal/app/<name>/` с доменной моделью (сущности, типы, ошибки) и интерфейсом репозитория.
2. Создайте use-case (сервис) в `internal/app/<name>/service.go`, используя интерфейс репозитория.
3. Создайте адаптер репозитория в `internal/app/<name>/adapters/postgres/repository.go` (GORM-реализация). GORM-модель с тегами определяется в адаптере, не в домене.
4. Создайте адаптер транспорта в `internal/app/<name>/adapters/connect/handler.go` (Connect-go хендлер).
5. Зарегистрируйте зависимости в `internal/bootstrap`: создание repo → service → handler, регистрация хендлера на mux.
6. Добавьте SQL-миграцию в `migrations/` (goose format) и proto-описание в `api/`.
7. Убедитесь, что `go build ./...` и `go test ./...` проходят успешно перед коммитом.
