# Hotel Concierge Bot — Базовая доменная модель и admin-интерфейс

## Контекст

На этом этапе необходимо заложить **базовую доменную структуру проекта без бизнес-логики**.

Требуется:
- Создать основные модели
- Определить связи между сущностями
- Добавить базовые enum-статусы
- Подготовить минимальную административную часть для внутреннего использования

Дополнительно необходимо:
- Подготовить тестовую основу
- Убедиться, что структура RSpec используется корректно
- Создать первую фабрику
- Добавить базовые примеры unit- и integration-тестов

Цель этапа — получить **рабочий каркас домена и административного интерфейса**, на который можно безопасно опираться в следующих фазах.

---

## Цель

Реализовать базовую доменную модель системы, настроить связи между сущностями, добавить необходимые enum-статусы, создать минимальный `admin` namespace с простым доступом и подготовить базовую тестовую структуру.

---

## Definition of Done

После выполнения:

- В проекте существуют все ключевые модели
- Между моделями настроены базовые ассоциации
- Enum-статусы описаны на уровне моделей
- Существует минимальный административный интерфейс без полноценной системы аутентификации
- Тестовая инфраструктура готова для дальнейшей разработки

---

## Ограничения

- **Не реализовывать бизнес-логику**
- **Не добавлять service objects**, кроме тестовой заглушки для примера unit-теста
- **Не добавлять Devise** или любую полноценную систему аутентификации
- Использовать только **простую заглушку авторизации** для admin-доступа
- Следовать **Rails-конвенциям** (именования, структура файлов, маршруты)
- Не переусложнять admin-интерфейс — только **минимальные страницы списков**
- **Не добавлять новые зависимости**
- Не реализовывать сложные валидации — только структура, связи и enum
- Все изменения должны укладываться в **scope текущего этапа**

---

## Ожидаемый результат

### Доменные модели

Созданы модели:

- `Hotel` (name, timezone)
- `Guest` (hotel_id, room_number, name, identifier_token)
- `Staff` (hotel_id, role, name, email)
- `Department` (hotel_id, name)
- `Conversation` (guest_id, status)
- `Message` (conversation_id, sender_type, content)
- `Ticket` (guest_id, department_id, status, priority)
- `KnowledgeBaseArticle` (hotel_id, title, content, category, published)

Дополнительно:
- Для `guests.identifier_token` добавлен **индекс**

---

### Ассоциации

Настроены базовые связи:

- `Hotel`
    - has_many :guests
    - has_many :staff
    - has_many :departments
    - has_many :knowledge_base_articles

- `Guest`
    - has_many :conversations
    - has_many :tickets

- `Conversation`
    - has_many :messages

- `Ticket`
    - belongs_to :department
    - belongs_to :staff *(если подключается на этом этапе)*

---

### Enum-статусы

Добавлены enum:

- `Ticket.status`
    - new
    - in_progress
    - done
    - canceled

- `Ticket.priority`
    - low
    - medium
    - high

- `Conversation.status`
    - open
    - waiting_for_staff
    - waiting_for_guest
    - closed

- `Staff.role`
    - admin
    - manager
    - staff

---

### Admin-интерфейс

Создан `admin` namespace:

- Реализован `Admin::BaseController`
- Добавлена **простая заглушка авторизации** (без Devise)

Созданы базовые страницы:

- Список отелей
- Список сотрудников
- Список заявок (tickets)

---

### Тестовая структура

Подготовлена базовая инфраструктура:

- Есть первая фабрика:
    - `Hotel`

- Добавлен пример **unit-теста**:
    - для service-заглушки

- Добавлен **integration test**:
    - например, для health check или root endpoint
