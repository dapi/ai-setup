Инициализируй проект на Rails 8 + Inertia.js + React.
В качестве референса и шагов ориентируйся на эти статьи:
- https://evilmartians.com/chronicles/inertiajs-in-rails-a-new-era-of-effortless-integration
- https://evilmartians.com/chronicles/simplicity-vanished-solving-the-mystery-with-inertia-js-and-rails

Задача по шагам:
- инициализируй проект, в качестве аргументов используй базу данных как Postgresql, при инициализации ты должен пропустить инициализацию следующих компонентов: js, asset pipeline, solid, git, test. Проект назови video_chat_and_translator. Проект должен создаться в текущей директории.
- Сделай все базовые настройки согласно референсным статьям. Используй те же библиотеки, что указаны в статье: tailwindcss, vite, react, typescript.
- Создай в корневой директории директорию docker, добавь туда Dockerfile и docker-compose.yml файл. Используй последнюю стабильную версию Ruby.
- Создай простую лендинговую страницу.
- Запусти проект и убедись что он работает.
- Если проект не работает, ты должен прочитать логи в докере, проанализировать их и изменить.

Требования:
- По поиску решения преимущественно пользуйся поиском в интернете. А также используй MCP инструмент Context 7.
- Для проверки работы используй инструмент playwright-cli. Ты должен сделать скриншот лендинговой страницы и сохранить его в корневой директории проекта и предоставить мне. Ты также должен убедиться, что на скриншоте именно та лендинговая страница, которую ты создавал.
- Ты занимаешься задачей, пока не будут решены все пункты и ты не предоставишь мне скриншот лендинговой страницы.

Ограничения:
- Ты не должен понижать версию Ruby on Rails и Ruby в том числе. 
- Ты не должен запускать проект локально
- Ты не должен понижать какие-либо версии, будь то tailwind, vite, react, typescript. Для решения проблемы пользуйся интернетом или MCP инструментами.
- Когда у тебя появляются сомнения по поводу какого-либо шага, ты должен остановиться и спросить у меня.
