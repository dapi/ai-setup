# Feature Brief: Add Linter & Formatter

**Issue:** akoltun/ai-setup#1
**Status:** Open
**Date:** 2026-04-05

---

## Problem

Проект находится на старте: исходный код уже содержит файлы с разным форматированием и без единых правил статического анализа. Без фиксации стиля на раннем этапе дальнейший код будет накапливать расхождения, усложняя ревью и увеличивая технический долг. Необходимо добавить механизмы статического анализа и автоматического форматирования до того, как кодовая база начнёт активно расти.

---

## Overview

Установить единый стиль кода и обеспечить возможность автоматических проверок качества через статический анализ и форматирование. Инструменты определены стеком проекта: Oxlint (линтер), Oxfmt (форматтер).

---

## Goals

- Обеспечить единый стиль кода во всём проекте
- Запуск проверок единой командой (`bun run lint`)
- Линтер охватывает файлы: `.js`, `.mjs`, `.cjs`, `.ts`, `.mts`, `.cts`, `.jsx`, `.tsx`
- Форматтер охватывает файлы: `.js`, `.jsx`, `.ts`, `.tsx`, `.json`, `.jsonc`, `.json5`, `.yaml`, `.toml`, `.html`, `.css`, `.scss`, `.less`, `.md`, `.mdx`

---

## Non-Goals

- CI/CD integration (out of scope for this issue)
- Pre-commit hooks
- Editor plugin configuration

---

## Acceptance Criteria

1. `bun run lint` завершается с кодом выхода 0 (без ошибок и предупреждений) на исходных файлах проекта
2. `bun run lint` применяет безопасные автоисправления и форматирует файлы перечисленных типов из раздела Goals
3. Конфигурационные файлы инструментов зафиксированы в репозитории
