# Plan: issue #9

## Цель
Оформить в корне репозитория каноническую папку `.prompts` с шаблонами для feature-workflow и убрать дублирование со старыми заготовками в `memory-bank/prompts/`.

## Шаги реализации
1. Создать `/.prompts/README.md`.
   В README зафиксировать:
   - что `.prompts` является каноническим местом для prompt-шаблонов;
   - соглашение вызова `prompt_name issue_number`;
   - краткое описание каждого шаблона;
   - правило использования единого плейсхолдера `issue_number`.
2. Создать обязательные шаблоны в `/.prompts/`:
   - `start_task`;
   - `rw_brief`;
   - `mk_spec`;
   - `rw_spec`;
   - `mk_plan`;
   - `rw_plan`;
   - `gnd_plan`;
   - `implement`.
3. Для каждого шаблона сделать инструкции самодостаточными:
   - указать входные данные;
   - указать конкретные пути файлов в `memory-bank/features/{issue_number}/`;
   - указать ожидаемый результат работы;
   - для review-шаблонов указать критерии проверки;
   - для generate-шаблонов указать, какой файл нужно создать или обновить.
4. Убрать старые заготовки из `memory-bank/prompts/`, чтобы не оставлять второй источник истины.
5. Проверить итоговую структуру репозитория:
   - `.prompts` существует в корне;
   - все обязательные шаблоны присутствуют;
   - старые prompt-файлы в `memory-bank/prompts/` отсутствуют;
   - содержимое шаблонов использует единый `issue_number`.

## Проверка
- Просмотреть дерево `.prompts`.
- Прочитать все новые шаблоны и убедиться, что:
  - они ссылаются только на существующую структуру репозитория;
  - используют `memory-bank/features/{issue_number}/`;
  - не смешивают `{task_number}` и `{task-number}`;
  - не требуют несуществующего runner или инструмента.
