**Задача:**  
Сделать механизм управления списками источников для сбора данных

---

**Проблема:**  
В системе отсутствует способ управления источниками данных, из-за чего любые изменения (добавление, редактирование, удаление) требуют изменения кода и деплоя.

---

**Для кого:**  
Внутренние сервисы и разработчики через API (на этапе MVP — без аутентификации)

---

**Контекст:**  
Feedium агрегирует данные из различных источников (Telegram, RSS и др.).  
С ростом количества источников увеличивается необходимость частых изменений, что усиливает зависимость от разработки

- Количество источников будет измеряться от 10 до 1000
- Частота изменений низкая
- Типа источников заранее известны:
  - каналы в Telegram
  - группы в Telegram
  - RSS фиды с сайтов
  - Парсинг html сайтов
- обязательно валидация входных данных

---

**Ограничения:**  
- Не требуется реализация integration и e2e тестов на данном этапе
- Без аутентификации и авторизации на данном этапе
- Управление касается только конфигурации источников
- Не включает выполнение сбора данных
- Не включает проверку доступности источников
- Не требуется пользовательский интерфейс (только API)

---

**Критерий успеха:**  

Добавление, изменение и удаление источников не требует изменения кода и деплоя
Любой пользователь системы может через API:
- создавать источник данных  
- редактировать источник данных  
- удалять источник данных  
- получать список всех источников
