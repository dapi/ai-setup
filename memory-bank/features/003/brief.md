# Feature Brief: Добавить CI/CD

**Issue:** akoltun/ai-setup#3
**Status:** Active
**Date:** 2026-04-06

---

## Problem

В проекте отсутствует автоматизированная проверка качества кода при изменениях в репозитории. Разработчики могут вмержить ветку с ошибками линтера, проблемами компиляции, поломанной сборкой или непрошедшими тестами. Также нет контроля над уровнем тестового покрытия, что снижает уверенность в корректности вносимых изменений. README.md не отражает текущее состояние проекта.

---

## Overview

Настроить CI/CD-пайплайн, который автоматически проверяет качество кода при каждом push в ветку и при попытке слить ветку в основную. Добавить контроль тестового покрытия. Обновить README.md: описать проект и отобразить актуальные статусные значки (CI badges).

---

## Goals

- Автоматический запуск проверок при push в любую ветку и при создании pull request
- Проверки включают: линтинг, проверку типов (компиляция), сборку, тесты
- Merge недоступен, если хотя бы одна проверка завершилась с error или warning
- Тестовое покрытие рассчитывается автоматически
- Merge недоступен, если тестовое покрытие ниже 80%
- README.md содержит описание проекта
- README.md содержит CI badges, отражающие актуальный статус пайплайна

---

## Non-Goals

- Настройка окружения для деплоя (staging, production)
- Кэширование зависимостей и оптимизация времени CI (может быть добавлено отдельно)
- Pre-commit хуки
- Конфигурация редакторов

---

## Acceptance Criteria

1. При каждом push в ветку автоматически запускаются: `bun lint`, `bun compile`, `bun build`, `bun test`
2. Если хотя бы одна из команд завершается с ошибкой или предупреждением — merge ветки заблокирован
3. Тестовое покрытие рассчитывается в CI; если оно ниже 80% — merge заблокирован
4. README.md обновлён: содержит описание проекта и CI badges
